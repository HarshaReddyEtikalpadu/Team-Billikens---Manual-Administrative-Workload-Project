Architecture & Data Design

Three-Tier Architecture

We adopt a three-tier architecture for clarity and modularity:

  • Presentation Layer: React frontend (dashboard views, role-based UI).
  • Application / Analytics Layer: Analytics engine, recommendations, aggregations.
  • Data Layer: Storage for requests, logs, daily summaries (prototype uses synthetic CSV/JSON).

Data Flow

Requests → Workload logging → ETL (Python) → Aggregation (DailySummary) → Dashboard visualizations & recommendations.

ERD (Entities & Relationships)

Six core entities described below. (ER diagram placeholder)

  • Departments (1:M Admins, 1:M Requests)
  • Admins (M:1 Departments, 1:M Requests)
  • Requests (M:1 Departments/Admins/RequestTypes)
  • RequestTypes (1:M Requests)
  • WorkloadLog (M:1 Admins, M:1 Requests)
  • DailySummary (Aggregated from Requests)
