Project Charter

Project Objectives

  • Provide real-time visibility into administrative workloads across departments.
  • Reduce document verification times and errors through analytics-driven recommendations.
  • Deliver a presentation-ready dashboard and supporting documentation by Dec 2025.

Scope

Included: Dashboard UI, synthetic data generation (Jan 2024 - Dec 2025), analytics visualizations, recommendations engine prototype, documentation, and presentation deliverables.

Excluded: Integration with live production systems, student-identifiable real data, full RBAC enforcement beyond prototype.

Timeline

  • Data period: Jan 2024 - Dec 2025 (synthetic)
  • Project completion and presentation: Dec 2025

Deliverables

  • Interactive dashboard prototype (React + Recharts)
  • Documentation and technical report
  • Presentation materials (slides & demo)

Team Roles & Responsibilities

  • Member 1 — Project Lead & Backend Developer: data ingestion and APIs.
  • Member 2 — Data Engineer: synthetic data pipelines and ETL.
  • Member 3 — Full Stack Developer: integration and deployment.
  • Member 4 — Frontend Developer & UX Designer: UI and accessibility.
  • Member 5 — Data Analyst & Documentation Lead: analytics and reporting.

Success Criteria

  • Dashboard accurately shows key KPIs for the synthetic dataset.
  • Presentation-ready site and demo available for the 5-minute presentation.
  • Recommendations show plausible improvements (30–50% where applicable).
